Business context. 

Greenfield Local hub is a well-established	 cooperative of local farmers and food producers, they offer many things such as a single location retail outlet for locally produced food and drink, they serve as a collective marketing for local farmers and food producers as well as product management, stock control, and product traceability. They are interested in digitalisation to keep existing customers happy and to engage with new ones. Our client is looking for a digital solution to allow customers to view the locally sourced products with transparent pricing and availability, to be able to place orders or delivery and to manage their accounts and view their order history. 

 

I am proposing to develop a new interactive and initiative website that will include all features that the client has required and some more in the following passages i will be justifying and describing my proposal. 

 

The functional and non-functional requirements of the system.  

 

Functional.  

 

Home Page. 

The homepage will have a navigation bar at the top of the page which will have links to the different pages included within the website, it will also have a main section and a footer. Within the main section there will be images and lesser amounts of information about the client. Within the footer there will be essential information about copyright and contact information from the client.  

 

No data inputs needed for this page. 

 

When the user clicks on the links included within the navigation bar they should be redirected straight to the desired page for example, when clicking on the about page they should be taken to the about page. 

 

Navigation bar will include. 

Home page  

Account page. 

About page  

Product page  

Log in page  

Sign up page 

 

 

Sign up page. 

Within the sign-up page it will allow the user to input their data to create an account such as email, username and password. Once the data has been inputted then the user signs up their data will be stored in a database so it will allow the users to use these details to log in within the future. 

 

Data inputs are required for this page. 

 

Log in page. 

Within the log in page, it will allow the user to use data they have inputted before within the sign-up page. For example, their username and passwords. Once these have been inputted, they will be allowed to have full access of the website. 

 

Data inputs are required for this page. 

 

Account page. 

Within the account page the user will be able to see their data and through this page it will allow them to be able to logout of their accounts. Users should be able to see their orders and purchases and their past ones.  

 

Product page. 

Within the product page the users will be able to see the products and their prices and a short description about them. Such as what they are and if there is enough time the ingredients list.  

 

About page. 

Within this page the users will be able to see the details and information about the client as well as the producers they collaborate with. In this page it will show the location of the outlet so customers will be able to visit and buy locally sourced produce. 

 

Non-functional requirements. 

 

Security 

The system should be secure and robust  

System must validate all user inputs 

System must use secure cookies 

System must use encryption for passwords.  

 

Maintainability. 

 It is important that after deployment the system should still be maintainable at a cost-effective state. 

To ensure maintainability i will be using good code practises through the development of the website i will ensure the naming conventions are relevant and easy to follow to allow for less confusion when taking future development. 

 

Performance. 

The system should load pages within 1 second. 

Latency: it shouldn't take more than 200 milliseconds for the system to respond to the user. 

 

Usability and accessibility. 

The prototype should follow all WCAG (Web Content Accessibility Guidelines) rules and be adapted due to different user needs. 

The system should be straightforward to use, both for the end user and the content editors that oversee the backend systems.

Description of the proposed solution. 

In the paragraphs above i was decomposing and analysing and setting requirements for the system. In this section i will be summarising the proposed solution for the digitalisation of the greenfield local hub and its main benefits to the client. 

 

Key features and benefits.  

Inclusive soloution: 

The website is going to be an inclusive platform to allow pre-existing and new users to navigate through all the information such as products, prices information about the produce, events, the retail outlet and where the client is located. This will allow users to navigate through the website and have access to all the information they will need this way it will allow the client to be assured that the users will be fully satisfied and up-to-date with all the new produce and news about events in real time, 

 

Quick and easy purchases.  

One of the most important features is that the users can use the proposed solution to purchase and order produce online making for a smooth and efficient transaction without them worrying about travel as it can be delivered to them or if they wanted, they can purchase it for pickup.  

 

Info blocks. 

The solution will provide the users with information about the different availability and pricing of the different produce and where it has come from as well providing information about the producer, their methods and the benefits of buying locally so they can be informed of what help they are doing.
